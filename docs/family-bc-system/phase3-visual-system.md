# Phase 3 Family B/C Visual System

Phase 3 closes the gap between the Workspace Agent workflow and Remotion implementation. It adds reusable, calibration-only visual foundations for Family B/C reels without starting a production reel.

## Operating Rules

- Family B/C remains primary: light editorial stage, chart-native proof, logo/icon actors, mobile-readable text, and CTA residue.
- Family A remains a secondary motion-energy reference only.
- Workspace Agent owns Lottie discovery, role fit, license notes, and human approval.
- Codex owns deterministic Lottie ingestion and rendering after approval.
- Lottie assets support route pulses, proof bursts, highlights, and CTA accents; they do not become hero metaphors.
- Review stills and contact sheets come before draft MP4s. Phase 3 blocks final MP4 rendering.

## Calibration Boundary

`FamilyBCVisualSystemCalibration` is a non-production composition. It validates the stage, chart, actor, Lottie, proof, and CTA components with fixture data only.

Phase 4 is the first approved pilot handoff stage. It requires source locks, route approval, asset approval, styleframes, and a Codex-ready packet before implementation.
