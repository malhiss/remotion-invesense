# Family B/C System Audit

## Verdict

Family B and Family C are the house benchmark. Family A is a motion/asset-energy reference, not the default visual style.

The current project is technically wired, but the existing `InvesenseCalibrationReel` is a dark Family A-style wiring test. It proves that Remotion, schemas, tests, build, and contact sheets work. It is not a visual foundation for production Invesense reels.

No old visual code is migrated into this project. The old repos are useful because they contain failure evidence, source-proof discipline, packet patterns, analogy libraries, proof operators, and review gates. They are not implementation sources.

## What Carries Forward

- Workspace Agent remains the creative director.
- Codex remains implementation-only.
- Source claims and chart proof must be audited before route labels lock.
- Family B/C light editorial grammar is the target.
- Family A contributes motion verbs: snap, route, glow, impact, latch, reveal, handoff.
- Remotion best practices are implementation source of truth.
- LottieFiles is an asset source for supporting motion, not a substitute for concept work.

## What Does Not Carry Forward

- Old visual components.
- Old labs and renders.
- Old geometry locks.
- Old dark dashboard shells.
- Old failed scene implementations.
- Old generic proof cards.
- Any visual route that requires labels to explain the mechanism.

## Benchmark Read

Family B reels are light-background chart explainers. They rely on chart proof, kinetic text, direct callouts, watermarks, and keyword CTA endings. The chart is usually the proof object, not wallpaper.

Family C reels are light-background logo and infographic explainers. Logos, icons, servers, rails, documents, payment objects, and institutional silhouettes behave as actors in the mechanism.

Family A reels are useful because they show stronger object motion and asset energy. We should borrow motion behavior, not the dark shell.

## Key Failure Diagnosis

The old project did not fail because Remotion was the wrong tool. It failed because Codex was asked to invent premium visual taste from prose. That produced technically valid videos that were visually weak, too dashboard-like, too diagrammatic, or too dependent on floating labels.

The new system must prevent that by forcing approved visual targets, source proof, chart role, asset board, Lottie policy, and human gates before Codex implementation.

## Implementation Consequence

This repo should become a creative operating system, not a template generator. It should provide contracts, packets, primitives, QA, and gates while preserving custom route selection per reel.
