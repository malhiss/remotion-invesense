# Family B/C Review And QA Gates

## Required Review Artifacts

- Hook still.
- First chart reveal still.
- Proof marker still.
- Logo or asset event still.
- CTA and residue still.
- Vertical contact sheet.
- Side-by-side benchmarkComparison sheet when a benchmark target is named.
- Source-chart comparison still when faithful recreation is required.

## Technical Gates

- `npm test`
- `npm run lint`
- `npm run build`
- review utility dry run
- review utility still/contact-sheet export

## Visual Gates

- Light editorial Family B/C grammar is clear.
- Chart proof is faithful when proof-critical.
- Bars scale by data, not style.
- Labels are sparse, attached, and mobile-readable.
- Lottie accents support motion texture only.
- CTA inherits mechanism residue.
- Final render approval is explicit.

## Stop Conditions

- Dashboard, SaaS UI, Canva spacing, slide-deck pacing, generic progress bars, or detached proof cards.
- Source chart distorted or replaced by placeholder proof.
- Lottie becomes the hero without approval.
- Proof appears before the visual event or source reveal.
- Final CTA could be swapped onto another topic.
