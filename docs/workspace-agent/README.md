# Workspace Agent Operating Pack

This pack is the Phase 1 knowledge layer for the Invesense Remotion project.
It turns the benchmark research, Remotion audit, Lottie policy, and old failure
lessons into operating rules for the Workspace Agent.

## Purpose

- Keep Workspace Agent as the creative director.
- Keep Codex as implementation-only.
- Force source locks, benchmark grounding, visual targets, asset decisions, and
  human gates before Remotion implementation.
- Prevent the old failure pattern: prose handoff directly to generic Codex visuals.

## Files

- `operating-manual.md`: role split and non-negotiable workflow rules.
- `family-grammar.md`: Family B/C primary grammar and Family A secondary motion use.
- `financial-mechanism-atlas.md`: finance terms mapped to force, constraint, state change, and metaphor families.
- `motif-proof-atlas.md`: motifs, wow events, and proof operators.
- `lottie-research-playbook.md`: LottieFiles search and approval rules.
- `codex-handoff-rules.md`: what a valid Codex handoff must contain.
- `human-gates.md`: required approvals before implementation and rendering.
- `benchmark-atlas.json`: every-video benchmark atlas.
- `benchmark-index.json`: search index by family, motif, asset, mechanism, and proof operator.

Phase 1 does not authorize production reel implementation.
