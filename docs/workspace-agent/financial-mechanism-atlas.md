# Financial Mechanism Atlas

The Workspace Agent starts with mechanism diagnosis, not keywords.

| Finance term | Force | Constraint | State change | Metaphor families |
| --- | --- | --- | --- | --- |
| support | repeated catch | price cannot fall cleanly | rebound or break | floor, shelf, moving average, safety line |
| resistance | repeated block | upside is capped | rejection or breakout | ceiling, wall, barrier, threshold |
| liquidity | movement capacity | channel width or depth | smooth flow or blockage | pipe, tank, route, rail |
| spread | distance between values | two values diverge | gap widens or compresses | bracket, caliper, dual rail |
| valuation compression | shrinking room | price multiple tightens | squeeze or release | clamp, chamber, press |
| fee drag | hidden loss | value leaks over time | balance drains | leak, siphon, drain |
| settlement | completion | transaction not final yet | proof locks | stamp, latch, endpoint |
| tokenization | access transformation | asset is locked or illiquid | units become transferable | vault, bridge, mint |
| screening | include or reject | criteria gate | pass/reject branch | scanner, sieve, gate |
| drawdown | support failure | floor breaks | asset falls | cracked floor, shaft |
| recovery | reclaim | asset must climb back | level regained | pulley, rebound, shelf reclaim |
| ranking | ordered comparison | magnitude must stay honest | leader emerges | strict bars, podium, lanes |
| policy transmission | force propagation | nodes pass pressure | effect travels | route chain, relay, rail |

Output requirement: every route must state force, constraint, state change, proof consequence, and mechanism family before visual design begins.
